# 開発ガイドライン

このプロジェクトの開発に参加する際のガイドラインです。詳細は `development_rules.md` を参照してください。

## 作業フロー

1. **Issue作成**: 作業前に必ずIssueを1つ作成する
2. **ブランチ作成**: `feature/<短い要約-issue番号>` または `fix/<短い要約-issue番号>` でブランチを作成
3. **実装**: 小さくコミット（Conventional Commits形式）
4. **PR作成**: 1人でもPRを作成してセルフレビュー
5. **マージ**: レビュー後、mainブランチにマージ
6. **CHANGELOG更新**: マージ後に変更履歴を更新

## コミットメッセージ形式

```
<type>(<scope>): <短い説明> (#<issue番号>)
```

**type例**: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`

**良い例**:
```
feat(calculation): マクロ栄養素配分計算機能を追加 (#1)
fix(form): 目標体重入力バリデーション修正 (#2)
docs(readme): セットアップ手順を追加 (#3)
```

## ブランチ命名規則

- `feature/<要約-issue番号>`: 新機能追加
- `fix/<要約-issue番号>`: バグ修正
- `docs/<要約-issue番号>`: ドキュメント更新

## PRテンプレート

```
目的/背景:
変更点:
動作確認:
影響範囲:
関連:
```

## コードスタイル

- TypeScriptの型定義を活用
- 関数/クラスには役割を明記したコメント
- 複雑なロジックには意図をコメントで説明

## テスト

主要な機能にはテストを追加してください。
